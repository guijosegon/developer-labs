# developer-labs

Repositório pessoal de estudos, experimentos e documentação técnica.

## Estrutura

```
developer-labs/
├── docs/                  # Anotações, conceitos e referências
│   ├── concepts/          # Conceitos teóricos e fundamentos
│   ├── cheatsheets/       # Referências rápidas
│   └── architecture/      # Diagramas e decisões arquiteturais
├── labs/                  # Experimentos práticos
│   ├── docker/            # Containers, Dockerfile, Compose
│   ├── kubernetes/        # Pods, Deployments, Helm, etc.
│   ├── security/          # AppSec, redes, pentest, criptografia
│   ├── react/             # Componentes, hooks, patterns
│   └── dotnet/            # ASP.NET Core, APIs, EF Core
├── projects/              # Mini-projetos completos
└── resources/             # Links, livros e cursos recomendados
```

## Tópicos em estudo

| Área | Tópicos |
|---|---|
| Infraestrutura | Docker, Kubernetes, CI/CD |
| Segurança | OWASP, TLS, JWT, pentest básico |
| Frontend | React, TypeScript, state management |
| Backend | ASP.NET Core, REST, gRPC |

## Como usar

Cada pasta tem seu próprio `README.md` com anotações, exemplos e links relevantes.
Os experimentos em `labs/` são independentes — entre na pasta e siga as instruções locais.
